Synchronization¶

Click here to see the source code

Brief Description: This aspect implements a thread execution for selected functions.

Detailed Description:

By using an aspect which implements the thread controller interface, an thread exclusive execution per function is guaranteed. With the use of a mutex (at least for the two default thread controllers) any thread will wait until the previous one finished executing the function. There will be no performance loss, if the thread interface is not implemented, but this aspect sill used.

threadsafe_execution (Aspect)¶

Brief Description: This advice wraps the intended function execution in a mutex lock.

acp::ignore_synchronized (Attribute)¶

Brief Description: This attribute can be used to exclude certain join points form being affected by this aspect.

Detailed Description:

If a scope already is annotated with this aspect’s attribute, individual join points within can be excluded with this attribute.

acp::synchronized (Attribute)¶

Brief Description: Triggers an synchronized call to every marked join point.
